# **AppRouter + XSUAA Authentication Flow**

## **Architecture Overview:**

```
User Browser
    ↓
AppRouter (Reverse Proxy + Auth Gateway)
    ↓
XSUAA (OAuth2 Authorization Server)
    ↓
Backend Services / HTML5 Apps
```

---

## **Step-by-Step Flow:**

### **1. User Accesses Application**
```
https://incture-inc-itm-saas-itm-saas-client1-approuter.cfapps.eu10-004.hana.ondemand.com
```
- User clicks your app URL
- Request hits **AppRouter**

### **2. AppRouter Checks Session**
- AppRouter checks if user has valid **JWT token** in session
- **No token?** → Redirect to XSUAA for login

### **3. Redirect to XSUAA Login**
```
https://inc-itm-saas-6zb5gllv.authentication.eu10.hana.ondemand.com/oauth/authorize
  ?client_id=sb-cw-itm-xsuaa!b703287
  &redirect_uri=https://incture-inc-itm-saas-itm-saas-client1-approuter.cfapps.eu10-004.hana.ondemand.com/login/callback
  &response_type=code
```

**AppRouter constructs OAuth2 authorization request:**
- `client_id`: From XSUAA binding credentials
- `redirect_uri`: Where to send user after login (**MUST be in xs-security.json**)
- `response_type=code`: Authorization Code flow

### **4. User Logs In (Identity Provider)**

XSUAA redirects to configured **Trust Configuration IdP**:
- **SAP ID Service**: Default SAP login
- **Custom IdP**: Your company's Identity Authentication Service (IAS)

User enters credentials → IdP validates

### **5. IdP Returns to XSUAA**

IdP sends authentication assertion back to XSUAA with user identity

### **6. XSUAA Issues Authorization Code**

XSUAA redirects back to AppRouter with **authorization code**:
```
https://incture-inc-itm-saas-itm-saas-client1-approuter.cfapps.eu10-004.hana.ondemand.com/login/callback
  ?code=abc123xyz
```

### **7. AppRouter Exchanges Code for Token**

AppRouter makes **backend call** to XSUAA:
```
POST https://inc-itm-saas-6zb5gllv.authentication.eu10.hana.ondemand.com/oauth/token
  client_id=sb-cw-itm-xsuaa!b703287
  client_secret=<from binding>
  code=abc123xyz
  grant_type=authorization_code
  redirect_uri=<same as before>
```

### **8. XSUAA Returns JWT Token**

XSUAA validates and returns:
```json
{
  "access_token": "eyJhbGc...",
  "refresh_token": "...",
  "id_token": "...",
  "expires_in": 43200
}
```

**JWT contains:**
- User identity (email, name)
- Scopes/roles
- Tenant ID (multi-tenancy)
- Expiry time

### **9. AppRouter Creates Session**

- AppRouter stores JWT in **session cookie**
- Redirects user to original requested path
- Cookie sent with all subsequent requests

### **10. Routing Requests**

AppRouter uses **xs-app.json** routes:

```json
{
  "source": "^/(.*)$",
  "target": "$1",
  "authenticationType": "xsuaa",  // Requires valid JWT
  "destination": "html5-apps-repo-rt"
}
```

**For each request:**
- AppRouter validates JWT (signature, expiry)
- Extracts scopes/roles
- Forwards request with JWT in `Authorization: Bearer <token>` header
- Backend services validate JWT for authorization

## **Multi-Tenancy Flow:**

### **Tenant-Specific Routing (TENANT_HOST_PATTERN)**

```yaml
TENANT_HOST_PATTERN: "^(.*)-${default-uri}"
```

**For subscriber tenant "acme":**
```
https://acme-incture-inc-itm-saas-itm-saas-client1-approuter.cfapps.eu10-004.hana.ondemand.com
     ↑
   Tenant subdomain
```

1. AppRouter extracts tenant ID: `acme`
2. Requests tenant-specific XSUAA token with tenant context
3. JWT includes `zid` (zone ID) for tenant isolation
4. Backend services read tenant from JWT and serve tenant data

## **Key Configuration Files:**

### **xs-security.json** (XSUAA Config)
```json
{
  "xsappname": "cw-itm-xsuaa",
  "tenant-mode": "shared",  // Multi-tenant
  "oauth2-configuration": {
    "redirect-uris": [
      "https://*.cfapps.eu10-004.hana.ondemand.com/**"  // MUST match callback URLs
    ]
  }
}
```

### **xs-app.json** (AppRouter Routes)
```json
{
  "routes": [
    {
      "source": "^/callback/v1.0/(.*)$",
      "authenticationType": "none"  // SaaS provisioning callbacks
    },
    {
      "source": "^/(.*)$",
      "authenticationType": "xsuaa"  // Protected routes
    }
  ]
}
```

### **mta.yaml** (Service Bindings)
```yaml
requires:
  - name: cw-itm-xsuaa  # AppRouter gets XSUAA credentials
```

## **Why Your Login Failed:**

**Error:** `redirect_uri does not match`

**Cause:**
```json
// xs-security.json is missing callback pattern
"redirect-uris": [
  "https://*.cfapps.eu10-004.hana.ondemand.com/**"  // ❌ Not deployed yet
]
```

**XSUAA received:**
```
redirect_uri=https://incture-inc-itm-saas-itm-saas-client1-approuter.cfapps.eu10-004.hana.ondemand.com/login/callback
```

**XSUAA rejected** because it doesn't match any configured redirect-uri.

---

## **Fix Steps:**

1. ✅ Update xs-security.json with correct redirect-uris
2. ✅ Change XSUAA to `managed-service` in mta.yaml
3. 🔄 **Rebuild & Redeploy** to update XSUAA service
4. ✅ Login will work!

**The key:** XSUAA must be updated with new redirect-uris before login works! 🎯
